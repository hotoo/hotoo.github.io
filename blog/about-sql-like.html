SQL LIKE 查询的一个问题

在做用户管理的时候，用户表(users) 有一个字段用来存储用户所在部门的信息， 字段的值一般是这样的： 

支付宝-技术部
支付宝-技术部-数据仓库

我们的系统一个根据用户所在部门进行过滤的功能，所以我们用了类似下面的查询语句： 

SELECT * FROM users WHERE dept LIKE "支付宝-技术部%";

这样的性能还好，但是今天发现一个问题是，当有两个部门名称很相似，比如： 

支付宝-CEO
支付宝-CEO办公室

这时在查询 支付宝CEO 的时候就会把 支付宝-CEO办公室 的用户也查询出来。

于是想到正则表达式： 

SELECT *
FROM users
WHERE REGEXP_LIKE(dept, "^支付宝-CEO$")
OR REGEXP_LIKE(dept, "^支付宝-CEO-[\w\-]+");

杯具的性能问题。

突然想到 

SELECT *
FROM users
WHERE dept="支付宝-CEO"
OR dept LIKE "支付宝-CEO-%";

于是简单有效的解决了这个问题。
